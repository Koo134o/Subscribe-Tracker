## 🌸 1. Business Purpose

### 🔹 1.1 Project background
<p align="center">

현대 사회는 유료 구독 서비스가 전 산업에 걸쳐 부상하고 있다. 이들은 OTT(넷플릭스, 디즈니+), 음원 스트리밍(스포티파이, 유튜브 뮤직), AI(Chat GPT, Claude), 각종 클라우드 및 유료 소프트웨어 등 매우 다양한 분야에서 비교적 저렴한 가격에 유용한 기능을 제공한다. 위 통계에서 확인할 수 있듯 현대 사회에서 많은 사람들이 구독 서비스를 사용하며, 여러 개의 서비스를 동시 구독하는 경우도 결코 적지 않다. 이용하는 서비스의 종류가 늘어남에 따라 소비자의 고정 지출 관리 또한 복잡해진다. 이 과정에서 본 프로젝트의 필요성을 느끼게 되었다.

  먼저, “이번 달에 총 얼마를 구독료로 지출하는가?” 라는 의문을 해결할 수 있도록 한다. 각 서비스마다 결제 시점과 결제 수단, 카드사 등이 분산되어있는 경우 사용자가 자신의 고정 지출 규모를 한눈에 파악하기 어렵다. 이러한 문제는 합리적인 소비를 방해하는 요소가 되므로, 기존 금융 서비스들과 차별점을 두어 파편화된 개별 구독 서비스들의 결제일과 금액을 명확하게 통합하여 보여주는 기능을 도입한다. 
  “이런 걸 결제했었나?”, “무료 체험하고 해지하려고 했는데 깜빡했네.”와 같은 문제도 빈번히 일어난다. 여러 개의 서비스를 동시에 사용해 정신이 없어 헷갈리거나, 무료 체험 프로모션을 통해 결제 등록 후 해지 타이밍을 놓치는 경우도 허다하다. 결제일 이전에 미리 알림을 받고 사용자 본인의 구독 현황을 실시간으로 확인할 수 있다면 이러한 불필요한 경제적 손실 또한 크게 줄일 수 있을 것이라고 생각하였다.
  
  위와 같은 일상적인 문제점들을 배경으로 하여 사용자의 관점에서 고정 지출의 투명성을 높이고, 관리의 번거로움을 해결할 수 있는 서비스를 개발하고자 한다. 본 프로젝트 'Subscribe Tracker'는 단순히 정보를 나열하는 기능 뿐만 아니라, 사용자가 수신하는 결제 알림 문자나 이메일 텍스트 데이터를 분석하여 자동으로 정보를 등록해주는 스마트 기능을 지향한다. 이를 통해 사용자는 모든 항목을 직접 입력해야하는 수동 방식의 불편함에서 벗어나, 더욱 편리하고 체계적인 구독 생활을 영위할 수 있을 것이라고 기대한다.

### 🎯 1.2 Goal
- 파편화된 여러 구독 정보를 하나의 리스트 형태로 통합
- 텍스트 분석 기반 자동 등록으로 과정 간소화
- 결제일 사전 알림 시스템으로 불필요한 지출 방지

### 👤 1.3 Target
- 다수의 구독 서비스를 사용하거나 해지 타이밍을 자주 잊어 불필요한 지출 경험자

## 🌸 4. Concept of Operation

### 1) Automated Subscription Registration
| | |
| :--- | :--- |
| **Purpose** | 사용자가 일일이 입력하는 번거로움을 제거하고 데이터 등록의 누락을 방지함 |
| **Approach** | Java의 정규표현식(Regex)을 활용하여 SMS/이메일 텍스트에서 서비스명, 금액, 날짜 데이터를 파싱(Parsing) |
| **Dynamics** | 외부 결제 알림 데이터가 시스템에 입력되거나 인식되는 시점 |
| **Goals** | 사용자 개입을 최소화한 실시간 자동 구독 데이터베이스 구축 및 관리 편의성 증대 |


### 2) Manual Entry & Data Correction
| | |
| :--- | :--- |
| **Purpose** | 자동 인식이 불가능한 특이 케이스나 누락된 데이터를 보완하여 가계부의 완전성을 기함 |
| **Approach** | 파싱 실패 시 사용자에게 알림을 보내고, 직접 입력할 수 있는 전용 UI 폼(Form) 제공 |
| **Dynamics** | 자동 등록 오류 발생 시 혹은 사용자가 '직접 추가' 메뉴를 선택할 때 |
| **Goals** | 시스템의 기술적 한계를 사용자의 개입으로 보완하여 데이터의 100% 정확도 달성 |


### 3) Multi-Step Smart Notification (D-3, D-1)
| | |
| :--- | :--- |
| **Purpose** | 무료 체험 종료나 미사용 서비스의 자동 결제로 인한 불필요한 경제적 손실 방지 |

| **Dynamics** | 결제 예정일 3일 전과 1일 전 정해진 시각(예: 오전 10시)에 시스템이 자동 트리거 |
| **Goals** | 사용자가 결제 전 충분히 고민하고 해지 여부를 결정할 수 있는 골든타임 확보 |

## 🌸 5. Problem Statement

### ⚠ Technical Difficulties (예상 문제)
* **비정형 텍스트 파싱의 복잡성**: 각 카드사나 결제 플랫폼마다 알림 문자(SMS) 및 이메일의 데이터 규격이 제각각이기 때문에, 하나의 정규표현식(Regex)만으로는 모든 결제 수단을 완벽하게 파싱하는 데 기술적 한계가 존재함.
* **실시간 외부 API 의존성**: 환율 계산 기능을 위해 외부 API를 호출할 때 네트워킹 지연이나 API 제공 측의 서버 장애가 발생할 경우, 실시간 금액 변환 기능이 정상적으로 작동하지 않을 위험이 있음.
* **백그라운드 스케줄링 정확도**: 모바일 환경이나 OS의 배터리 최적화 정책에 따라 시스템 스케줄러가 결제 예정일 알림(D-3, D-1)을 정확한 시간에 트리거하지 못할 가능성을 고려해야 함.
* **데이터 일관성 유지**: 사용자가 자동 등록된 내역을 수동으로 수정하거나 삭제할 때, 이 변경 사항이 즉각적으로 통계 데이터 및 월간 리포트에 오류 없이 반영되도록 동기화 로직을 정교하게 설계해야 함.

### ⚙ Non-Functional Requirements (비기능적 요구사항)
* **데이터 무결성 및 보안**: 사용자의 결제 내역은 민감한 금융 정보에 해당하므로, 데이터베이스 저장 시 암호화를 철저히 하고 사용자 간의 데이터가 물리적/논리적으로 완벽히 격리되어야 함.
* **시스템 응답 속도 및 가용성**: 대량의 구독 내역을 조회하거나 복잡한 통계 차트를 렌더링할 때 사용자에게 지연 없는 쾌적한 인터페이스를 제공해야 하며, 시스템은 상시 가용 상태를 유지해야 함.
* **사용자 경험(UX)의 직관성**: 다양한 연령대의 사용자가 별도의 매뉴얼 없이도 자동 등록 및 알림 설정 기능을 손쉽게 이용할 수 있도록 직관적인 UI 디자인을 유지해야 함.
* **확장성 있는 아키텍처**: 향후 새로운 구독 카테고리가 추가되거나 결제 수단이 다양화되더라도 전체 시스템의 구조적인 변경 없이 유연하게 기능을 확장할 수 있는 모듈화된 설계를 지향함.

## 🌸 6. Glossary
* **Parsing (파싱)**: 문자열 형태의 비정형 데이터(결제 알림 문자/이메일)를 시스템이 이해할 수 있는 정형 데이터 구조로 분해하고 추출하는 과정. 카드사별로 상이한 알림 문구에서 [금액], [날짜], [업체명]을 정확히 식별하는 본 프로젝트의 핵심 기술임.
* **Regex (Regular Expression, 정규표현식)**: 특정한 규칙을 가진 문자열의 집합을 표현하기 위한 형식 언어. 다양한 결제 플랫폼의 알림 패턴을 하나의 알고리즘으로 처리하기 위한 매칭 규칙 및 슬라이싱 도구로 활용됨.
* **CRUD (Create, Read, Update, Delete)**: 소프트웨어가 데이터를 처리하는 네 가지 기본 기능. 구독 정보의 신규 등록(C), 목록 조회(R), 정보 수정(U), 서비스 해지 후 삭제(D)로 이어지는 구독 경제의 전체 생애주기 관리를 의미함.
* **D-Day Alert (결제일 사전 알림)**: 데이터베이스에 저장된 차기 결제 예정일을 기준으로, 실제 결제가 이루어지기 3일 전(D-3)과 1일 전(D-1)에 사용자에게 주의 환기 메시지를 보내 지출 여부를 재결정하게 돕는 자동화 시나리오.
* **Deep-link (딥링크)**: 알림 메시지 클릭 시 앱의 메인 화면이 아닌, 해당 구독 서비스의 '해지 페이지'나 '상세 설정' 화면으로 사용자를 즉시 랜딩(Landing)시키는 연결 기술로 사용자 편의성을 극대화함.
* **Currency Conversion (환율 변환)**: 유튜브 프리미엄이나 넷플릭스 등 해외 플랫폼 결제 시 발생하는 외화($ 등)를 실시간 환율 API 데이터를 기반으로 원화(KRW)로 환산하여, 사용자가 실제 체감하는 지출 비용을 통일된 단위로 제공하는 기능.
* **Data Integrity (데이터 무결성)**: 자동 파싱이나 사용자의 수동 수정 과정에서 데이터가 중복되거나 누락되지 않고, 항상 정확하고 일관된 상태를 유지해야 하는 시스템의 품질 특성.
* **SHA-256 (Secure Hash Algorithm)**: 사용자의 비밀번호와 같은 민감한 개인정보를 안전하게 보호하기 위한 단방향 암호화 알고리즘. 보안 사고 시에도 원본 데이터 유출을 방어하여 서비스의 신뢰도를 높임.
* **Look & Feel (룩앤필)**: 사용자 인터페이스(UI)의 색상, 폰트, 레이아웃 등이 주는 시각적 인상과 사용자가 체감하는 조작감. 본 프로젝트에서는 화이트/다크 모드 전환 기능을 통해 최적의 시각적 경험을 제공함.
* **Fallback Logic (폴백 로직)**: 자동 등록 과정에서 파싱 에러나 데이터 인식 실패가 발생했을 때, 시스템이 멈추지 않고 '수동 입력 모드'로 전환하여 데이터의 연속성을 보장하는 예외 처리 메커니즘.
